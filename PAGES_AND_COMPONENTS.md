# План сайта: перевозка и сопровождение в Китай для лечения зубов

**Цель:** привлечь посетителей и собрать заявки на сопровождение (только фронтенд, без бэкенда и БД).

---

## Основные возможности (ключевые функции)

1. **Главный экран с призывом к действию** — понятное описание услуги и кнопка «Оставить заявку».
2. **Форма заявки на сопровождение** — имя, контакт, даты поездки, комментарий; отправка (пока без бэкенда — показ сообщения «Заявка отправлена»).
3. **Информация об услуге** — что входит в сопровождение, этапы поездки, преимущества лечения в Китае.
4. **Блок доверия** — отзывы/истории (статичный контент), контакты, возможно кратко «о нас».
5. **Навигация и футер** — переход между разделами и к форме заявки, контакты, соцсети (если есть).

---

## Список страниц

| № | Страница        | URL (пример) | Назначение |
|---|-----------------|--------------|------------|
| 1 | Главная         | `/`          | Входная точка: описание услуги, преимущества, призыв к заявке |
| 2 | Услуги / Как это работает | `/uslugi` или `/#uslugi` | Подробно: что входит, этапы, цены (если показываете) |
| 3 | Оставить заявку | `/zayavka` или `/#zayavka` | Форма заявки на сопровождение |
| 4 | Контакты        | `/kontakty` или `/#kontakty` | Телефон, мессенджеры, адрес/режим работы (при необходимости) |

**Вариант «одна страница» (лендинг):** все блоки на одной странице с якорными ссылками: Главная = Услуги + Заявка + Контакты. Тогда страница одна — `/`, разделы: герой, услуги, заявка, контакты.

Рекомендация: начать с **одной страницы (лендинг)** — проще и достаточно для привлечения и сбора заявок.

---

## Вариант A: Одна страница (лендинг)

Одна страница `/` с секциями (якоря: `#uslugi`, `#zayavka`, `#kontakty`).

### Компоненты страницы

| Секция        | Компоненты | Что видит пользователь |
|---------------|------------|-------------------------|
| **Шапка**     | `Header`   | Логотип/название услуги, навигация: Услуги, Заявка, Контакты. На мобиле — бургер-меню. |
| **Герой**     | `Hero`     | Заголовок («Сопровождение в Китай для лечения зубов»), короткий подзаголовок, кнопка «Оставить заявку» (скролл к форме). Фоновое изображение по желанию. |
| **Услуги**    | `Services` или `HowItWorks` | Заголовок секции. Карточки или список: что входит (встреча, трансфер, переводчик, клиника, проживание и т.д.), кратко этапы поездки. Текстовый контент — статичный. |
| **Преимущества** | `Benefits` | Заголовок. 3–4 пункта: например «Качество», «Цена», «Комфорт», «Без языка». Иконки + короткий текст. |
| **Форма заявки** | `RequestForm` | Заголовок «Оставить заявку». Поля: имя, телефон, email, желаемые даты поездки (или «предпочтительный период»), комментарий. Кнопка «Отправить». После «отправки» — сообщение «Спасибо, мы свяжемся с вами» (без реальной отправки). |
| **Отзывы**    | `Reviews` (опционально) | Заголовок «Отзывы». 2–3 статичных отзыва: имя, текст, дата. Без слайдера — просто карточки. |
| **Контакты**   | `Contacts` | Телефон, WhatsApp/Telegram, email. Можно адрес или «онлайн-запись». |
| **Подвал**     | `Footer`   | Копирайт, повтор ссылок (Услуги, Заявка, Контакты), контакты или соцсети. |

### Данные на странице (статичные, без БД)

- Тексты секций (заголовки, описания) — в коде или в константах/переменных.
- Пункты услуг и преимуществ — массивы строк или объектов `{ title, description }`.
- Отзывы — массив `{ name, text, date }`.
- Контакты — телефон, email, ссылки на мессенджеры.

## Рекомендация по старту

- Сделать **одну страницу (Вариант A)** с секциями и якорями.
- Компоненты: `Header`, `Hero`, `Services`, `Benefits`, `RequestForm`, `Contacts`, `Footer`. Опционально — `Reviews`.
- Все данные хранить в константах в коде (массивы текстов, контакты). Форма при отправке только показывает «Заявка отправлена», без бэкенда.
- Позже при появлении бэкенда — заменить обработчик формы на реальный запрос и при необходимости разнести контент по страницам (Вариант B).

## Краткая сводка компонентов

| Компонент     | Где используется | Данные |
|---------------|------------------|--------|
| `Header`      | Все страницы / весь лендинг | Название, пункты меню |
| `Hero`        | Главная / верх лендинга | Заголовок, подзаголовок, текст кнопки |
| `Services`    | Секция «Услуги» | Список услуг/этапов |
| `Benefits`    | Секция «Преимущества» | Заголовок, массив преимуществ |
| `RequestForm` | Секция «Заявка» / страница заявки | Поля формы; после отправки — сообщение |
| `Reviews`     | Опционально | Массив отзывов (имя, текст, дата) |
| `Contacts`    | Секция / страница контактов | Телефон, email, мессенджеры |
| `Footer`      | Внизу страницы | Копирайт, ссылки, контакты |
